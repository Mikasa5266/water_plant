# 后端开发规范草案

最后更新：2026-05-25

后端当前只预留 Python 入口，尚未选择具体框架和依赖管理方案。

## 1. 后续需要确定

- Python 版本。
- Web 框架，例如 FastAPI。
- 依赖管理方式。
- Lint、format、test 命令。
- 配置读取方式。
- Mock 数据位置。
- Agent 编排边界。

## 2. 目录边界

后端业务代码统一放在：

```text
backend/app/
```

不要把 Python 服务代码放到 `frontend/`、`docs/` 或仓库根目录。

## 3. 接口约束

- 新接口先更新 `contracts/openapi.yaml`。
- 请求和响应字段必须有清晰类型。
- 设备 ID、告警 ID、Agent run ID 等标识必须稳定。
- 不在接口中返回真实硬件敏感信息。

## 4. 安全约束

- 不提交 `.env`。
- 不提交真实密钥、数据库密码、客户内网地址、PLC/SCADA 连接信息。
- 当前阶段不实现真实硬件写控制。
- 后续如果需要控制真实设备，必须单独设计权限、审计、模拟/真实环境隔离。
