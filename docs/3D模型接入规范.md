# 3D 模型接入规范

> 适用对象：3D 建模师 → 前端开发 交付
> 前置阅读：[建模协作规范](./MODELING_GUIDE.md)
> 最后更新：2026-05-27

---

## 1. 文件格式

| 格式 | 支持程度 | 推荐度 | 说明 |
|------|---------|--------|------|
| **GLB** (glTF Binary) | 原生支持 `useGLTF()` | ⭐⭐⭐ **推荐** | 单文件包含几何+纹理+动画，Three.js 生态最佳 |
| **glTF** (.gltf + .bin + 纹理) | 原生支持 `useGLTF()` | ⭐⭐ | 多文件不方便管理 |
| OBJ | 需额外 Loader `OBJLoader` | ⭐ | 不含材质，需 MTL 文件，加载器不稳定 |
| FBX | 需额外 Loader `FBXLoader` | ⭐ | 格式老旧，FBXLoader 解析经常出错 |
| STL | 需额外 Loader `STLLoader` | ✘ | 纯几何无材质，不适合此项目 |

**结论：所有模型统一导出为 `.glb`**。

## 2. 导入路径

模型文件放入前端 `public/` 目录，前端通过相对路径引用：

```
frontend/public/models/
├── supervisor_hub.glb
├── dosing_module.glb
├── uf_module.glb
├── ro_module.glb
├── pump_module.glb
└── (其他资源)
```

前端代码引用：

```tsx
import { useGLTF } from '@react-three/drei';
const { scene } = useGLTF('/models/dosing_module.glb');
```

## 3. 单位与比例

| 参数 | 规范值 | 说明 |
|------|--------|------|
| **长度单位** | 1 单位 = 1 米 | Three.js 默认，无需缩放 |
| **模型比例** | 1:1 真实尺寸 | 设备实际 2m 高 → 模型 2 单位高 |
| **坐标手系** | 右手系 (Y-up) | glTF 标准，Blender/Maya 默认 |
| **前方向** | -Z (Three.js 默认) | 导出前确认模型朝向 |

**关键约定**：

```
Three.js 坐标系：X=右, Y=上, Z=前(屏幕外)
场景坐标映射：Three.js [x, y, z] = 数据坐标 [x, z, y]
```

## 4. 模型原点 (Pivot Point)

| 设备类型 | 原点位置 | 原因 |
|---------|---------|------|
| 静态设备（药箱、膜池、超滤柱） | 底面中心 | 放在地面上不需要额外偏移 |
| 旋转设备（泵、阀门、搅拌器） | 旋转轴中心 | `useFrame` 绕 origin 旋转 |
| 监管中枢 | 底面中心 | 光环绕 Y 轴旋转 |

**错误示例**：泵体原点在几何中心 → 旋转时会"晃"而不是"转"。

## 5. 面数与性能

| 模型类型 | 面数上限 (三角形) | 说明 |
|---------|-------------------|------|
| 大型设备（中枢、膜池） | **3 万面** | 视觉焦点，可略微精细 |
| 中型设备（药箱、超滤柱） | **1.5 万面** | 常规设备 |
| 小型设备（泵、阀门、管路） | **5 千面** | 数量多，严格控制 |
| 装饰细节（护栏、螺栓） | **2 千面** | 非焦点 |

**场景总面数**：建议控制在 **15 万面以内**（5 个设备模块合计）。

## 6. 纹理规范

| 参数 | 规范值 | 说明 |
|------|--------|------|
| **纹理分辨率** | 1024×1024 或 2048×2048 | 非游戏项目，不需要 4K |
| **格式** | PNG (不透明) / JPEG (照片级) | 嵌入 GLB 中 |
| **最大单文件** | 5 MB | 超出需拆分或压缩 |
| **UV 展开** | 必须 | 无 UV 的模型无法贴纹理 |

**建议**：优先用程序化材质（PBR metallic-roughness），纹理仅用于 logo、标签、仪表盘等需要精细细节的部位。

## 8. 代码集成示例

当前每个设备模块采用"几何体与动画解耦"架构：

```tsx
// ─── 当前（几何体占位）───
<mesh ref={pumpRef}>
  <cylinderGeometry args={[2, 2.2, 4.5, 16]} />
  <meshStandardMaterial color="#475569" roughness={0.3} metalness={0.7} />
</mesh>

// ─── 换成 GLB 模型（仅改一行）───
const { scene } = useGLTF('/models/pump_module.glb');
<primitive object={scene} ref={pumpRef} scale={1} />
```

**接入要点**：
- `ref` 保持不变，`useFrame` 旋转/动画代码完全不动
- `scale={1}` 前提是模型按 1:1 米制导出
- GLB 模型的材质继承自文件，前端不覆盖

## 9. 导出清单 (建模师交付)

交付模型时请附以下信息：

| 交付项 | 说明 |
|--------|------|
| `.glb` 文件 | 放在 `public/models/` 下 |
| 设备名称 + 文件对应表 | 见第 7 节 |
| 单位确认 | 是否 1 单位 = 1 米 |
| 原点确认 | 是否底面中心 / 旋转轴中心 |
| 面数 | 实际三角形数量 |
| 已知问题 | 穿透、法线翻转、动画异常等 |

## 10. Blender 导出设置（参考）

如果使用 Blender 建模，导出时确保：

```
格式: glTF 2.0 (.glb)
格式选项:
  ✅ 二进制 .glb
变换:
  ✅ Y 轴朝上
几何数据:
  ✅ 应用修改器
  ✅ 三角化面
  ✅ 压缩
动画:
  ✅ 烘焙全部动作（如有）
```
