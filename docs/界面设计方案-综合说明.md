# 未来水厂智能操作系统 — 界面设计方案

> 设计日期：2026-05-26
> 参考文档：`规划书-界面设计相关摘录.md`
> 技术栈：React + React Three Fiber（3D引擎），TypeScript

---

## 一、总体布局

采用 **OS 桌面操作系统范式**（非传统大屏），分为四个区域：

```
+------------------------------------------------------------+
|  顶栏：系统名称 + 状态指示（系统在线/5智能体/实时时钟）     |
+----------+-------------------------------+------------------+
| 应用坞    |                               |  信息面板         |
| (左侧dock)|      3D 水厂场景视口          |  (右侧)          |
|            |    (React Three Fiber)      |                  |
| 监管智能体 |   顶部覆盖层：告警数/产水/健康 |  当前代理         |
| 超滤智能体 |                               |  当前思考         |
| 反渗透智能体|   底部覆盖层：流程时间轴       |  决策链           |
| 加药智能体 |                               |  事件日志         |
| 泵组智能体 |                               |                  |
+----------+-------------------------------+------------------+
|  底部任务栏：首页 · 窗口标签 · 通知图标 · 时钟               |
+------------------------------------------------------------+
```

### 1.1 左侧 Dock（应用栏）

- 竖向排列 5 个智能体图标（监管/UF/RO/加药/泵组）
- 每个图标带状态指示灯（绿/黄/红）
- 点击图标 → 打开/切换对应 Agent 窗口
- 异常时图标自动闪烁、显示角标数字

### 1.2 中央 3D 视口

- 全尺寸 React Three Fiber 3D 场景，支持旋转/缩放
- 顶部覆盖层：3~4 个概览指标（总告警、产水负荷、健康度）
- 底部覆盖层：5 步骤流程时间轴（异常感知→数据上送→AI分析→智能体分发→执行恢复）

### 1.3 右侧信息面板

- 当前代理：显示正在操作的智能体名称
- 当前思考：AI 推理过程摘要（与 3D 场景中的思考气泡同步）
- 决策链：当前处理的步骤序列
- 事件日志：带时间戳的操作记录

### 1.4 底部任务栏

- 左侧"首页"按钮：点击切换回主界面（当用户在某个 Agent 窗口时，点击可快速返回 OS 桌面）
- 中间显示已打开的 Agent 窗口标签（可切换）
- 右侧系统托盘（通知图标、时间）

## 二、3D 场景布局

### 2.1 空间定位

```
          [加药模块]                          [反渗透模块]
    药箱 · 泵 · 阀门                   膜组A · 膜组B · 膜组C · 循环泵
        \                                  /
         \    [超滤模块]    [监管智能体]   /
          \ 超滤柱×3 · 阀组    ★         /
           \____________   _____________/
                        | |
                   [泵组智能体] · 泵组
```

- **监管智能体**：场景中心，发光蓝色圆柱体+光环（大脑中枢）
- **加药模块**：左前方 − 药箱（圆柱体）、加药泵、阀门
- **超滤模块**：左后方 − 3 个超滤柱、阀组
- **反渗透模块**：右前方 − 3 个膜组、循环泵
- **泵组模块**：右下方 − 泵组设备
- 各模块与监管之间用 **管道连接**，异常时粒子沿管道飞行

### 2.2 Agent 模型

- **监管智能体是特殊中枢**，非普通 Agent 球体。它以大型发光蓝色圆柱体 + 悬浮光环（torus）呈现在场景几何中心，代表大脑中枢
- **其他 4 个专项 Agent**（加药/UF/RO/泵组）以**发光球体**表示，悬浮于各自设备模块上方
- 专项 Agent 有颜色区分（加药=琥珀、UF=绿、RO=橙、泵=紫），自身闪烁表示正在工作
- 监管中枢的状态通过发光强度/光环旋转速度/颜色变化表达
- 每个 Agent（含监管）头顶可弹出**思考气泡**（AI 正在分析时）

### 2.3 设备模型

- 第一阶段用基础几何体（box/cylinder/sphere/torus）占位
  - 但**所有 3D 动画效果（闪烁、粒子飞行、设备旋转、视角聚焦等）仍需完整实现**
  - 动画逻辑与模型解耦，后续替换 .glb/.gltf 真实模型时动画可直接复用
- 暂不导入大模型

## 四、粒子动画 & 数据传输

### 4.1 推荐方案：管道 + 粒子飞行

采用 **方案A（带管道连接）** 为主，辅以粒子飞线效果：

- 场景中静态布置管道（连接各设备模块与监管智能体）
- 数据传输时，粒子沿管道路径飞行（红色/橙色/绿色）
- 粒子颜色语义：
  - 🔴 **红色粒子** = 异常数据上报（设备→监管）
  - 🟠 **橙色粒子** = 任务派发（监管→专项Agent）
  - 🟢 **绿色粒子** = 执行指令 / 恢复（Agent→设备）

### 4.2 3D 动画效果

| 动画类型  | 视觉表现                | 触发时机    |
| ----- | ------------------- | ------- |
| 异常闪烁  | 设备模型红色闪烁 (0.26s)    | 数据越限检测时 |
| 数据采集  | 红色粒子从设备沿管道飞向监管      | 异常检测确认后 |
| AI 思考 | Agent 发光旋转 + 思考气泡弹出 | AI 分析阶段 |
| 任务派发  | 橙色粒子从监管飞向专项Agent    | 决策完成后   |
| **视角聚焦** | **3D 摄像机平滑聚焦到目标设备，局部放大（持续时间约 2-3s）** | **专项 Agent 下发指令时** |
| 设备操作  | 阀门旋转、泵叶轮转动、液体流动     | 指令到达设备时 |
| **视角恢复** | **3D 摄像机平滑恢复到全局视图** | **设备操作完成后** |
| 恢复渐变  | 红→绿渐变，参数曲线回归基线      | 异常解除时   |

### 4.3 Agent 思考气泡

在 3D 场景中，当 Agent 处于"思考/分析"状态时：

- Agent 头顶弹出**半透明气泡卡**（深色背景 + 青色边框）
- 气泡带**三角形尾部**指向 Agent
- 内容以**打字机效果**逐条显示 AI 推理要点
- 气泡标题显示 "AI 正在分析..."
- 点击气泡 → 弹出 Agent 详情窗口（跳转到完整分析页面）

## 五、OS 通知系统

### 5.1 系统通知弹窗

- 异常发生时，桌面右上角弹出**系统通知弹窗**
- 左侧红色指示条 + 红色感叹号图标
- 标题：`系统异常告警`
- 描述：`加药区 · 泵频率异常升高`
- 时间戳：`15秒前 · 监管智能体已介入分析`
- 底部 "查看详情 →" 按钮 → 打开对应 Agent 窗口

### 5.2 通知触发流程

1. 系统检测到数据异常
2. 桌面右上角弹出通知（带淡入动画）
3. 底部任务栏通知图标闪烁
4. 点击通知 → 直接打开对应 Agent 分析窗口
5. 异常解除后 → 通知自动消失 / 变为恢复通知

### 7.2 实时数据契约（每个 Agent 4 个参数）

每条参数的结构统一为：
```typescript
interface MetricField {
  label: string                     // 显示名称
  value: number | string            // 初始值（IDLE 态下一定在正常范围内）
  unit: string                      // 单位
  normalRange: NormalRange          // 参考范围
  alarmRule: 'lower' | 'upper' | 'both' | null  // 越限触发方向
  /** 按键触发后，模拟引擎向此方向偏移，越过 alarmThreshold 后触发告警 */
  shiftDirection?: 'up' | 'down'    // 异常偏移方向（默认 up）
}

type NormalRange =
  | { min: number; max: number }    // 数值上下界：[0.3, 0.6]
  | { max: number }                 // 单上界：≤30
  | { min: number }                 // 单下界：≥95
  | string[]                        // 枚举值：["良好","注意","严重"]
  | null                            // 无范围约束
```

## 八、场景阶段枚举（ScenarioPhase）

场景/事件的生命周期由以下枚举驱动，作为唯一的主控状态源：

```typescript
enum ScenarioPhase {
  IDLE = 'idle',                     // 空闲监控中（背景动画持续）
  ANOMALY_DETECTED = 'detected',     // 设备越限 → 红色粒子上报
  SUPERVISOR_ANALYZING = 'analyzing',// 监管分析 → 思考气泡
  DISPATCHING = 'dispatching',       // 派发任务 → 橙色粒子
  AGENT_ANALYZING = 'agent_analyzing', // 专项Agent推演
  EXECUTING = 'executing',           // 下发指令 → 绿色粒子 + 视角聚焦
  DEVICE_OPERATING = 'operating',    // 设备操作动画 + 参数恢复
  RECOVERING = 'recovering',         // 视角恢复 + 红→绿渐变
  RECOVERED = 'recovered'            // 恢复完成
}
```

各阶段对应的粒子颜色和 3D 效果：

| 阶段 | 粒子 | 3D 效果 | 视角 |
|------|------|---------|------|
| IDLE | 无（背景粒子低速巡回） | 设备平稳运行 | 全局 |
| ANOMALY_DETECTED | 🔴 红色（设备→监管） | 设备红色闪烁 | 全局 |
| SUPERVISOR_ANALYZING | — | 监管中枢发光+思考气泡 | 全局 |
| DISPATCHING | 🟠 橙色（监管→专项Agent） | 派发路径可见 | 全局 |
| AGENT_ANALYZING | — | 专项Agent闪烁+思考气泡 | 全局 |
| EXECUTING | 🟢 绿色（Agent→设备） | 决策传递 | 聚焦到设备 |
| DEVICE_OPERATING | — | 阀门旋转/泵转动/参数恢复 | 聚焦保持 |
| RECOVERING | 🟢 绿色消退 | 红→绿渐变 | 平滑恢复全局 |
| RECOVERED | — | 所有恢复正常 | 全局 |

### 8.1 ScenarioPhase → UI 四态映射

| ScenarioPhase | Agent UI 状态 | 含义 |
|---------------|---------------|------|
| `IDLE` | 🟢 **正常** | Agent 运行正常，定时采集数据 |

> 切回规则：场景执行完毕后停留在 `RECOVERED` 态 **2 秒**，然后 `useScenarioStore` 自动将 phase 切回 `IDLE`。
> 空闲态下背景动画持续运行（设备微动、粒子低速巡回），看起来和真实系统完全一样。
> Esc 强制终止：从任何 phase 直接回到 `IDLE`（清除所有粒子/闪烁/聚焦状态）。
| `ANOMALY_DETECTED` | 🟡 **待处理** | 数据已越限，等待监管分析 |
| `SUPERVISOR_ANALYZING` | 🔴 **异常闪烁** | 监管智能体正在分析 |
| `DISPATCHING` | 🔴 **异常闪烁** | 任务派发中 |
| `AGENT_ANALYZING` | 🔴 **异常闪烁** | 专项 Agent 正在推演 |
| `EXECUTING` | 🔴 **异常闪烁** | 指令下发中 |
| `DEVICE_OPERATING` | 🔴 **异常闪烁** | 设备操作执行中 |
| `RECOVERING` | 🟢 **恢复** | 参数回归，视角恢复 |
| `RECOVERED` | 🟢 **恢复** | 异常已解除，即将回到正常 |

> 实现提示：`useAgentStatus.ts` 只需监听 `useScenarioStore.phase`，按上表映射到 `'normal' | 'pending' | 'alarm' | 'recovering'` 四态，各 UI 组件按 §3.2 四态映射表渲染。
> 监管中枢与专项 Agent 的 UI 状态同步跟随此映射，不单独维护状态。

## 九、交互流程（以加药异常为例）

```
[1] 加药泵频率异常升高 → 设备红色闪烁
    ↓ (红色粒子沿管道飞向监管)
[2] 监管智能体闪烁 → 思考气泡显示分析要点
    ↓ (用户可点击气泡查看详情)
[3] 监管决策 → 橙色粒子派发至加药智能体
    ↓
[4] 加药智能体闪烁 → 推演投加方案
    ↓ (思考气泡显示推演过程)
[5] 加药智能体下发指令 → 绿色粒子飞向设备
    ↓ **→ 3D 摄像机聚焦放大到加药模块（持续约 2-3s）**
[6] 阀门旋转、泵启动 → 参数曲线回归
    ↓ **→ 3D 摄像机平滑恢复到全局视图**
[7] 监管智能体显示"异常已处理" → 系统恢复
```

## 十、设计原则摘要

1. **OS 桌面范式** — 不是监控大屏，是 Windows/macOS 式桌面
2. **智能体 = 应用程序** — 每个 Agent 是独立窗口，可拖拽/缩放/最小化
3. **数据驱动动画** — 粒子颜色语义明确（红=异常/橙=派发/绿=执行）
4. **AI 思考可视化** — 3D 气泡 + 打字机效果 + 可点击跳转详情
5. **状态分层** — 设备级(闪烁) → Agent级(状态灯) → 系统级(通知)
6. **先几何体后模型** — 先用基础几何体搭建，再替换 .glb 真实模型

## 十一、演示触发方式

采用 **键盘快捷键触发**，不设显式的演示按钮，保持系统真实感。

### 11.1 快捷键映射

| 按键 | 触发事件 | 说明 |
|------|----------|------|
| `F1` | 加药异常场景 | 泵频率异常升高 → 加药智能体介入 |
| `F2` | 超滤异常场景 | 跨膜压差升高 → 超滤智能体介入 |
| `F3` | 反渗透异常场景 | 膜压差偏高 → 膜智能体介入 |
| `F4` | 泵组异常场景 | 电流过载 → 泵组智能体介入 |

### 11.2 辅助按键

| 按键 | 功能 |
|------|------|
| `?` 或 `F12` | 显示/隐藏快捷键帮助浮层（半透明面板，列出所有快捷键映射） |
| `Ctrl+Shift+D` | 打开开发者调试面板（可作为备份入口，包含手动触发事件的按钮） |
| `Esc` | 按优先级：关浮层 > 终止场景 > 关通知 > 最小化窗口 |

### 11.3 触发机制

- 按键触发后，数据经过**渐进模拟**（非瞬间跳变）：
  `正常数据 → 缓慢偏移（约 3s） → 超出阈值 → 触发告警`
  整个过程看起来像是系统"真的"检测到了异常，而不是瞬间切换场景
- 系统运行期间 3D 场景持续有**背景动画**（设备轻微运转、水流流动、数据粒子低速巡回），避免"静止等待"的尴尬
- 当前无异常时，系统处于"正常监控"状态，看起来和真实系统完全一样

### 11.4 交互规则

**首页按钮**
- 行为：关闭所有 Agent 窗口 → 回到 OS 桌面（3D 视口在前）
- 如果已在桌面 → 无操作
- 快捷键：`Ctrl+Home`

**窗口多例规则**
- **同一 Agent 不允许多开**（单例模式）
- 点击 Dock 图标时：未打开则打开 → 已打开则切换到前台
- 最大同时打开窗口数：**5 个**（每个 Agent 最多一个）

**最小化与恢复**
- 最小化：窗口缩到任务栏标签
- 恢复：点击任务栏标签 → 恢复到最后位置和尺寸
- 关闭窗口：Dock 图标状态灯不受影响（Agent 仍在后台运行）

**通知队列**
- 单条 Toast 显示，后一条**替换**前一条
- 多异常同时发生 → 合并为一条「N 个异常」通知
- 异常通知停留 **5 秒**自动消失，恢复通知 **2 秒**自动消失

**异常并发策略**
- 场景状态机为**单实例**（非队列），一次只执行一个异常场景
- 场景进行中（`phase !== IDLE`）再次按 F1~F4：**忽略**（不排队、不覆盖、不合并）
- 但支持「连续异常」：当前场景执行完毕回到 `IDLE` 后，下一次按键正常响应
- 如需强制打断：按 `Esc` → 当前场景终止 → 回到 `IDLE` → 再按新快捷键启动新场景
- 设计意图：避免多场景粒子/聚焦/动画重叠导致视觉混乱，保证每次演示清晰可跟

**Esc 行为（按优先级从高到低）**
1. 帮助浮层打开 → **关闭浮层**（场景不受影响）
2. 场景进行中（`phase !== IDLE`）→ **终止当前场景**，回到 `IDLE`
3. 通知弹出 → **关闭通知**
4. Agent 窗口打开 → **最小化当前窗口**
5. 无打开项 → **无操作**

**? 键冲突防护**
- 仅在**非输入态**下生效（`event.target.tagName` 排除 `<input>`、`<textarea>`、`[contenteditable]`）

## 十二、性能约束

### 12.1 目标帧率

| 场景 | 目标 FPS | 备注 |
|------|----------|------|
| 空闲监控（无异常） | ≥ 45fps | 持续有背景动画 |
| 异常事件进行中 | ≥ 30fps | 粒子+闪烁+聚焦同时运行时 |
| 最大负载（5窗口+3D+粒子） | ≥ 24fps | 所有特效全开 |

### 12.2 降级策略（LOD）

| 级别 | 触发条件 | 关闭效果 |
|------|----------|----------|
| 全特效 | 初始 | 所有效果开启 |
| LOD1 | FPS < 30 持续 2s | 关闭粒子尾迹，粒子数量减半 |
| LOD2 | FPS < 20 持续 2s | 关闭背景动画，仅保留关键闪烁 |
| LOD3 | FPS < 15 持续 2s | 关闭所有特效，仅保留静态 3D |

### 12.3 设备边界
- 桌面端：Chrome/Edge/Firefox 最新版本
- **不支持移动端**（第一阶段不考虑）
- 推荐配置：独立显卡 GTX 1060 或以上，16GB 内存
- 最低配置：集成显卡（近 3 年 Intel Iris / AMD Vega），8GB 内存

### 12.4 R3F 优化策略
- 使用 `@react-three/drei` 的 `Instances` 合并同类几何体
- 粒子系统用 `BufferGeometry` + `Points`，不用独立 mesh
- `useFrame` 动画用 `useRef` 避免 re-render
- 窗口切换时 3D 场景不 rebuild，仅通过 store 控制显隐
