# 分工方案

根据设计文档的目标和当前代码结构，核心思路是：**按功能模块垂直切分**，两人各自负责独立的文件和目录，最大限度避免同文件冲突。

## 人员 A：OS 桌面系统 + UI 层

负责所有 2D UI 组件、窗口系统、通知系统、状态面板：

| 任务 | 涉及文件 / 目录 |
|---|---|
| 左侧 Dock 栏（5 个 Agent 图标 + 状态灯 + 角标） | 新建 `src/components/Dock/` |
| Agent 窗口系统（拖拽 / 缩放 / 最小化 / 单例） | 新建 `src/components/AgentWindow/` |
| 右侧信息面板（当前代理 / 思考 / 决策链 / 事件日志） | 新建 `src/components/InfoPanel/` |
| 底部任务栏（首页按钮 / 窗口标签 / 通知图标 / 时钟） | 新建 `src/components/Taskbar/` |
| OS 通知系统（Toast 弹窗 / 队列 / 自动消失） | 新建 `src/components/Notification/` |
| 快捷键帮助浮层（`?` / `F12` 触发） | 新建 `src/components/HelpOverlay/` |
| 全局状态管理（Zustand store） | 新建 `src/stores/useScenarioStore.ts`、`src/stores/useSystemStore.ts`、`src/stores/useWindowStore.ts` |
| 键盘事件监听 + `Esc` 优先级逻辑 | 新建 `src/hooks/useKeyboard.ts` |

## 人员 B：3D 场景 + 动画引擎

负责 React Three Fiber 场景、粒子系统、摄像机动画、设备模型：

| 任务 | 涉及文件 / 目录 |

| 管道系统（静态管道 + 粒子路径） | 改造 `src/simulation3d/builders/pipelineBuilder.ts` |
| 粒子飞行系统（红 / 橙 / 绿三色语义） | 改造 `src/simulation3d/builders/particleAnimBuilder.ts` |

## 共享契约

在开始各自编码前，先一起定义这些接口文件：

| 文件 | 说明 |
|---|---|
| `src/stores/useScenarioStore.ts` | `ScenarioPhase` 枚举 + store 接口（A 写实现，B 只读消费） |
| `src/types/scenario.ts` | 共享类型定义（`ScenarioPhase`、`AgentId`、`AgentStatus` 等） |
| `src/stores/useSystemStore.ts` | 系统级指标接口（告警数 / 产水 / 健康度） |

**约定：**

- B 从 store 读取 `phase` 来驱动 3D 动画
- A 负责写入 `phase` 的状态转换逻辑
- 接口定好后，两人互不依赖

## 减少冲突的关键规则

- 不碰对方的目录：A 不改 `simulation3d/`，B 不改 `components/`
- `DashboardPage.tsx` 最后合并：各自开发完成后，由一个人统一把新组件组装进主页面
- 共享 store 接口先锁定：`types` 文件和 store 接口一旦定好，改动需要通知对方
- 各自建分支：A 用 `feat/os-desktop-ui`，B 用 `feat/3d-scene-engine`，最后合并到 `dev`
